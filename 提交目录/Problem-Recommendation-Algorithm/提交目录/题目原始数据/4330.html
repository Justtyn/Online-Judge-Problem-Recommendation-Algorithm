Description

    自从参加ACM比赛，小罗遇到了许许多多的电子资料，最常接触到的就是PDF文档。

用来浏览PDF文件的Adobe Reader软件提供了一种打印格式——小册子打印，能将一张纸的正反两面各分成两半共四部分，打印结束以后，把所有的纸从中间对折起来，变成了一个小册子，翻这个小册子，可以发现上的页码是连续的，但是是单独拿出一张纸来看发现上面的页码可能不是连续的。

假如打印的文档原来有8页，那么要把它打印成小册子就需要2张纸，两张纸的页码是这么分布的

    有一天，小罗发现一张纸，是以前打印的一个小册子上的，他想知道这张纸是那个小册子的第几张纸，小册子总共用了几张纸，但他不想人肉计算，请你编写一个程序，来告诉他答案。

设正面的左边页码为a，右边页码为b，反面的左边页码c，右边页码d。

保证b<c<d<a。

Input

每组输入一张纸某一面（可能是正面，也可能是反面）的左边页码a和右边页码b(0<a,b<2^28)

保证输入的页码是真实可能出现的，就算那个文档的页数不是四的整数倍，多出那几页也会标上页码，假如原来只有7页，那么打印后第8页所在的部分一样会写上页码，而且打印程序是尽可能省纸的。

a,b为0 0 时结束程序

Output

对于每组输入数据对应输出两个数，第一个数为这张纸所在小册纸的总用纸数，第二个数为这张纸在小册子上所在的位置（从页码1所在的纸开始数）。

Sample Input

8 1<br/>6 3<br/>2 7<br/>12 1<br/>10 3<br/>0 0

Sample Output

2 1<br/>2 2<br/>2 1<br/>3 1<br/>3 2
