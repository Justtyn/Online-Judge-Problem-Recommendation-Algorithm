Description

2008年3月24日，北京奥运火炬开始传递。为配合这一盛事，即时通讯公司Absolutely Cool Messenger (ACM)决定在自己的用户群中开展虚拟火炬传递活动。首先由公司随机抽取若干名用户为火炬传递第一棒选手。拿到火炬的用户需要在一分钟内决定要将自己的火炬传给哪些人。第k棒火炬手可以将火炬传给w-k+1个人，从而使他们成为第k+1棒火炬手，第w+1棒选手为最后一棒，不能再继续传递。w由ACM公司设置。活动过程中，所有人的状态不可知，当活动结束后，火炬手的id前将出现一个火炬图标。如果某用户将火炬传给了已经是火炬手的朋友，则认为这个机会就被浪费掉了。 BNUEP的学生多是ACM的忠实用户，并且为了能够在一分钟内准确地传递火炬，他们事先列出了一个传递火炬邀请的优先级名单，每个人的名单上都列出了w名朋友，按照优先级从前往后排列。如果他们能够传递给m个人，则他们将选择前m名朋友传递火炬。现在告诉你ACM公司初始选择了哪些用户作为第一棒选手，请你计算一下，活动结束后一共有多少名BNUEP的学生名字前出现火炬。

Input

有多组测试数据。输入第一行为一个正整数C，代表随后有C组数据。每组数据第一行有三个整数w，n，t，0 < w <= 6，为ACM公司设置的系数。0 < n <= 20000，为BNUEP学生的人数。0 <= t <= 100，为BNUEP被选上的第一棒选手的人数。同时，设学生的编号分别为1-n之间的连续正整数。每组数据的第二行有t个合法的学生编号，用空格分隔，为第一棒选手的编号。随后n行，其中的第i行有w个合法的学生编号，为i号学生选择的w名朋友编号，按照优先级从高到低排列，不会出现自己选择自己的情况。

Output

每组测试数据只用输出一行。即活动结束后一共有多少名BNUEP学生名字前面会出现火炬。

Sample Input

1<br/>2 6 2<br/>1 2<br/>2 3<br/>3 4<br/>4 5<br/>5 6<br/>6 1<br/>1 2<br/>

Sample Output

5<br/>

HINT

说明：系数为2，初始选中1、2号学生。然后1号传给了2、3，2号传给了3、4。这时2号学生已经是火炬手，所以浪费了一个机会。1、2号同时传给了3，所以3号也相当于浪费了一个机会。第二次每人只能传递给1个朋友。3号传给了4（浪费），4号传给了5。这时传递停止，同时5名学生成了火炬手。
