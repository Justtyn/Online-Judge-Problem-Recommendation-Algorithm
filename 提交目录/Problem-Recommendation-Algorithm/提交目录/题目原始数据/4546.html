Description

由于前段时间cc都很忙，而且总要在书房查很多资料，所以当现在有些空闲的时候 就要整理一下现在的书房，因为实在是太乱了，书堆的到处都是。当cc拿到线形代数在手里，随便翻看了一下，偶然看到里面的矩阵乘法的时候，忽然灵光一闪，想到如果矩阵相乘用计算机来实现的话，一定很快，很方便；而又进一步思考发现如果先乘哪一个， 后乘哪一个， 会在计算量上有很大的区别，而如果通过计算机来实现的话，可能就是成千上万倍的计算量。例如，有A1 : 10 * 100 , A2 : 100 * 5 , A3 : 5 * 50 ，这三个矩阵，如果先乘前两个( A1 * A2 )，再乘第三个，就一共需要计算7500次；而如果 先计算后两个( A2 * A3 )，就只需要75000次。哇，差拉好多，所以这看起来就复杂了，所以当cc 收拾完东西，找到聪明的你来帮助cc 一起来解决这一发现。

Input

有多组测试数据，每组先输入一个n ( 1 < = n < = 30 )，表示接下来有n 组的且相邻的矩阵Ai ( 1 < = i < = n ，从 1 到n 顺序输入) 的长 ai 和 宽bi ，当n 为0 时代表输入结束。(保证每组测试数据输入都是合法的，且在计算过程中和结果中所有数number，全部在 1 < = number < = 2^31 -1范围内)。

Output

输出占一行，输出给出的最少的相乘的次数。

Sample Input

3<br/>10 100<br/>100 5<br/>5 50<br/>0<br/>

Sample Output

7500
