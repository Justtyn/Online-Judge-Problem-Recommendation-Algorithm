2561: 下载 download

Time Limit: 1 Sec  Memory Limit: 128 MB
Submit: 0  Solved: 0
[Submit][Status][Web Board][Creator:][下载FPS1元][添加到购物车][下载测试数据1元][0kb]

Description

迅雷是个不错的软件（不是打广告……）但是 Jia 委员有一天想下冷门资源了，迅雷也帮不了他了。于是他找到了有迅雷会员的 Jiangzh 来帮他在 M 分钟之内下载完。现在有 N 个任务同时开始下载，这 N 个资源是整理过的，所以每个都是 A     MB 大小

（1MB=1024KB），由于网速比较稳定，每个任务有一个固定的下载速度 Bi   KB/s，不过有些下载速度 Bi 太慢了，Jiangzh 可以让所有任务加速 R KB/s 持续 M 分钟。

Input

第一行三个整数，M,N,R，每两个整数中间用一个空格隔开第二行一个整数 A 第三行 N 个整数 Bi，每两个整数中间用一个空格隔开

Output

输出一行一个浮点数，下载完成的最短时间（单位 s，保留两位小数），如果 Jiangzh 不能按时完成则输出“xi wen le jian”（红色加粗部分，不含引号）

Sample Input

[Sample 1]
10 3 200
100
100 250 20
[Sample 2]
5 3 200
100
100 250 20

Sample Output

[Sample 1]
465.45
[Sample 2]
xi wen le jian

HINT


对于 30%的数据：N<=1000，A<=1024 对于 100%的数据：N<=100000，M<=100000，A<= 1048576，Bi<=500，R<=maxlongint


样例1解释：有3 个任务要在10 分钟之内下载完，每个任务100MB(102400KB)，加上加速，每个文件的下载速度为300KB/s, 450KB/s,
220KB/s，三个任务下载完成的时间分别为



341.33s, 227.56s,
465.45s，所以全部下载完成需要465.45s。





样例解释2：5 分钟之内不能完成
