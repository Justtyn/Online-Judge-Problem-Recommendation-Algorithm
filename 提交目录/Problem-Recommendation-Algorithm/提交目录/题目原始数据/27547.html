27547: 交通灯

Description

　　Farmer John 的小镇里有M (1 <= M <= 14,000)条路，编号为 1..M ，同时有 N (2 <= N <= 300)
路口，编号为1..N。没有两条或两条以上的路连接相同的两个路口。也没有一条路连回出发点。.
路口i和路口j之间的通过时间为T_ij (1 <= T_ij <= 100) (从i到j和从j到i的时间是相同的. T_ij = T_ji).
　　每个路口只有一个交通灯,显示两种颜色:蓝色和紫色.两种颜色轮流显示一段时间. 只有两端的路口交通灯颜色一致,这条路才能走.在过一条路的过程中两端的交通灯是否一致已经不关事了。也就是只在乎出发的那一刻两个路口颜色是否一致。车允许在路口等。具体给出的数据如下：
　　N个路口，M条路。
　　每条路的两个路口编号，和过这条路的时间（来回时间一样）
　　每个路口在一个交通灯显示周期内，显示蓝色的时间 DB_i (1 <=DB_i <= 100) 和 显示紫色的时间 DP_i (1 <= DP_i <= 100)
　　每个路口交通灯初始颜色，并且会持续一段时间R_i (1 <= R_i <= 100) ，才开始执行该路口交通灯的显示周期。
　　输入出发点S (1 <=S <= N) 和目标点D (1 <= D <= N; D != S).


输入格式:
* Line 1: S and D
* Line 2: N and M
* 下来N行，每行的格式为: C_i, R_i, DB_i, and DP_i（其中C_i为初始灯的颜色）
* 下来M行，每行三个整数：i, j, and T_ij

输出格式:

一行一个整数，到达目标的最短时间，如果无法到达输出0.

样例输入:

1 4
4 5
B 2 16 99
P 6 32 13
P 2 87 4
P 38 96 49
1 2 4
1 3 40
2 3 75
2 4 76
3 4 77


样例输出:
127
样例最少的时间是127，路径是 1-2-4.

具体解释如下：一开始车在出发点1. 路口1 这时的颜色是蓝的，因为路口2的颜色为紫色,所以车在路口1 等了2s，然后向路口2出发，花了4s到达路口2.过了6s,路口2的灯变成蓝色，路口4还要再过32s才能变成蓝色，所以在等32s。32s后路口2变紫色，路口4变蓝色，所以还要再等13s。；路口2才能变蓝色，那个时候路口4和2统一颜色了，所以从路口2 出发向路口4走去，76s后到达路口4.
总共花的时间： 2+4+32+13+76=127 seconds.
