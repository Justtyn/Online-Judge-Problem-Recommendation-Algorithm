Description

总是有事情要做！这里是一个表格，表示你今天早上要做的事：

1. 做数学作业。
2. 打电话话给你的奶奶。
3. 打电话给爸爸妈妈。
4. 打电话给你的朋友。
5. 喂狗。
6. 出去溜狗。
7. 看电视。
（这是好的，你的父亲让你看电视，而不是一整天使用互联网。）

你的父母给你这些任务的约束：

1.在你看电视之前做好数学作业。
2.在你打电话给你的朋友之前做好数学作业。
3.在你打电话给你奶奶之前做好数学作业。
4.在你打电话给爸爸妈妈之前打电话给你的朋友。
5.在打电话给爸爸妈妈之后喂狗。

你的做表格上（以上）的事的顺序现在可以缩写为：

1,7
1,4
2,1
3,4
3,5

在完成后面任务前完成前面任务编号。
不幸的是，你要把一天另外的约束发送给你的父母。
写一个程序来输出你原来的要做的事和另外的约束，你的工作在命令你必须去做，或者，如果你不能完成这些，你应该输出” Cannot complete these tasks. Going to bed. (不能完成这些，我去睡觉) ”。

Input

你会得到两个数字，每行一个数，表示包含在你的原始做的事上面给出的另外的约束。
数据以0,0(两行)结束。你可以确定将最多10个有另外的约束。

Output

你应该输出一个任务列表的顺序，他们应该被执行，或者一个错误消息：说任务无法完成。
如果有多个任务可以完成，下面的提示即使打破规则也要使用：
如果有一组的任务可以同时执行的过程中，最小编号的任务应首先执行。

Sample Input

6
2
5
4
0
0

Sample Output

3 5 6 2 1 4 7

HINT

输入数据告诉我们，任务6之前必须完成任务2，任务5之前必须完成任务4。没有先决条件的任务是3和6，所以我们选择3最先执行，因为它在先决条件的任务中编号最小。5和6是可能的，5和6是可能的；5先选择，然后选6。下一个选2，其次是1。然后，4和7是可能的，编号最小的一个是首选。
