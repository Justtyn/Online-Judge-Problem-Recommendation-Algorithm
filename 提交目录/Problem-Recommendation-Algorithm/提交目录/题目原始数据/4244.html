Description

LLB是一个聪明的大盗，他曾经做了很多出色的案子，但警察却拿他没有办法，一直逍遥法外。这不，LLB又开始干一件大案子了。市郊有一座豪华别墅，是亿万富翁TQ买下的。LLB知道TQ在他的保险柜里藏了很多贵重的珠宝，很久以来就想造访他的别墅了。但是，一直没有找到合适的机会。经过漫长的等待，终于机会来了。这一天，TQ带着他全家出去环游世界了，整个别墅变得空无一人。LLB就开始大显身手了。他没费多大劲，就潜入了TQ的别墅，很快找到保险箱的位置。但是，怎样打开保险箱呢？LLB似乎遇到难题了。

 

TQ的保险箱比较奇怪，要打开它可不容易。好在LLB是个聪明的大盗啊，费了一番工夫，他就弄明白了保险箱的密码规则了。保险箱上有许许多多的按键，按键分布如图所示：

保险箱上总共有N行N列按键，行与行之间的距离相等，列与列之间的距离相等，并且行距与列距也相等。要打开保险箱，LLB首先要用线段把左下角的按键和其他所有按键连结起来，然后找出满足以下条件的按键：连结该按键和左下角按键的线段，只经过这两个按键，即在连结该按键和左下角按键的线段上，再没有别的按键出现。这些按键被称为有效按键。如图所示，图中标明了当N=6时，所有的有效按键及它们与左下角按键连结的线段。好了，这点小事，自然难不倒LLB。找出有效按键后，要打开保险箱，只差一步了。只要把所有有效的按键按照一定的顺序连续按一遍，保险箱自然就会打开。也就是说，有效按键的排列顺序，就是保险箱的密码，如果按的顺序不对，保险箱不会打开。当然，按键的顺序是TQ设定的，LLB可不知道。于是，LLB打算把所有可能的按键顺序都试一遍，他知道这样一定能找到正确的密码。

 

别忘了，LLB可是个聪明的大盗，他带了一个超级强大的智能机器人，他可以让机器人帮他试密码（而且这个机器人速度超级快的哦）。他打算在TQ的别墅里住下了，况且别墅还有很多好吃好喝的呢。不过LLB忽然产生一个古怪的念头，他知道在运气最坏的情况下，机器人要尝试的次数是一个很大很大的数，要算出这个数是一件很困难的事。但他现在只想知道这个数的末尾有多少个0。于是，他想到用计算机来帮他解决这个难题，现在你来帮帮他，编一个程序，告诉他答案吧。

 

Input

输入的第一行是一个整数T（1 ≤ T ≤300），表示有T组输入数据。 以下T行每行是一组测试数据，每组测试数据包含一个整数N（2 ≤ N ≤301），N表示保险箱上的按键的行数和列数。

Output

对每组测试数据，输出一行答案，即最坏情况下机器人所需尝试的总次数的末尾的0的个数。

Sample Input

3<br/>2<br/>5<br/>7<br/><br/>

Sample Output

0<br/>2<br/>6<br/><br/>

HINT

提示：想想什么情况下尾数会产生0 ？
