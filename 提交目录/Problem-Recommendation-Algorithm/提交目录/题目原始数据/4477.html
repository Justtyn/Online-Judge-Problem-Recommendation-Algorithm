4477: 曹操赤壁之战

Description

      三国时期，曹操带领兵马在赤壁战周瑜和诸葛亮，       曹操是著名的军事家，擅长排兵布阵，但赤壁之战，他知道对手周瑜和诸葛亮都是绝顶聪明的，曹操这次采用了一个新的战术，他将战场划分成50000列，每列依次编号为1, 2, …, 50000。然后，他把自己几十万的兵力也迅速均分为50000组(假设一定能均分)，同样也编号为1, 2, …, 50000，同时下命令，让编号为i的组迅速站在第i列(i = 1, 2, …, 50000)，形成了可怕的阵容。当周瑜，诸葛亮的兵力到达时，曹操会多次发命令，将大部分兵力集中在某几列上，集中兵力共同作战。合并指令为U i j，意思是让第i组现在所在的列中所有兵力作为一个整体接至第j组现在所在列的尾部。
  • 例如，假设现在i在第i列，j在第j列，Uij之后，i也在第j列了。
  • 再如：经过了很多合并后，假设现在i在a列，而j在b列，这时Uij后，整个a列(包括a列中除i组以外的其他组)就接到b列最后一组的后面(不一定是j组后面)。
      然而，周瑜早已在战略上取得了主动。在此之前，他已经布下秘密间谍，现在他可以随时得到曹操的调动命令。在曹操命令合并兵力的同时，诸葛亮为了及时了解当前曹操的兵力分布情况，会问周瑜：Q i j，意思是说，问曹操的第i组兵和第j组兵当前是否在同一列中，如果在同一列中，那么在该列中，i组和j组之间共有多少组兵(包括i,j两组)。       可是周瑜没诸葛亮那么聪明，没法快速回答诸葛亮发出的询问，现在要你来编写一个程序，帮助周瑜快速回答诸葛亮的询问。       赤壁之战，曹操失算了，主要是因为他发了一些让士兵不知所措的命令，就是已经在同一列的两组兵，曹操仍然发命令让他们合并，这时候士兵就乱了。所以，你的程序需要注意这点，不能让士兵出乱。

Input

      输入第一行为一个整数T,(1<=T<=500000),表示有T条命令。接下来T行，每行一条命令，命令有两种.
  • 曹操发出的命令，按顺序包含三个要素U,i,j ，中间用空格分隔，表示曹操下令让第i组所在列的所有兵力，按照头接尾的规则，接到第j组所在列的最后一个组后面。(最后一个组不一定是第j组，只是属于j现在所在的列的组)
  • 诸葛亮询问周瑜的命令，按顺序包含三个要素，Q,i,j，中间用空格分隔，表示诸葛亮问周瑜现在曹操的兵力，第i组和第j组是否处于同一列。

Output

曹操发的命令，你不需要输出任何内容。 对诸葛亮询问的命令，用单独的一行输出每条命令的结果。对于Qij,如果i,j已经在同一列，先输出YES，紧接着输出一个空格，然后输出在这列里，第i组和第j组之间有多少组兵(包括i组和j组)。如果i,j不在同一列，输出NO即可。

Sample Input

4
U 2 3
Q 1 2
U 2 4
Q 4 2

Sample Output

NO
YES 3
