4311: (2007新生赛) 12.9大合唱

Description

　　又是一年的12•9纪念日，各学院也组织起合唱队参加学校一年一度的大合唱比赛。信软学院的合唱队来了很多同学，老师准备给大家安排一下队形。 　　可是老师在给大家排队形的时候遇到一个问题。老师并不知道一共来了多少同学，但当他让同学们p1人一排时，多出来r1个同学；如果p2人一排，又会多出来r2个同学…… 　　就这样排了好几次都没成功。折腾了好久，老师觉得这样子太麻烦，不如再找来一些同学，让同学们站成一个方阵（行和列相等），这样就省事多了。 　　老师想知道最少需要再找多少同学才能使大家组成一个方阵。

Input

　　第一行输入一个整数T，表示一共有T组测试数据。　　每组测试数据第一行包含一个整数n (1<=n<=10)，表示老师一共让同学们排了n次队。接下来的n行，每行输入两个整数pi ri (1<=ri<=pi<=25)，表示此次排队pi个同学一排，并且多出来ri个同学。　　保证输入的pi两两互质。

Output

　　每组测试数据输出一行，输出一个整数s，表示最少需要找来s个同学就能组成方阵。　　注意，同学的总人数可能有多个解，你需找出最小的一个解，并输出对应的s。

Sample Input

4<br/>3<br/>2 1<br/>3 2<br/>5 1<br/>9<br/>2 1<br/>3 2<br/>5 3<br/>7 4<br/>11 5<br/>13 6<br/>17 7<br/>19 8<br/>23 9<br/>9<br/>2 1<br/>3 2<br/>5 4<br/>7 5<br/>11 5<br/>13 6<br/>17 8<br/>19 8<br/>23 8<br/>9<br/>2 1<br/>3 2<br/>5 3<br/>7 4<br/>11 4<br/>13 4<br/>17 8<br/>19 8<br/>23 9<br/>

Sample Output

5<br/>27466<br/>2915<br/>4753<br/>

HINT

　　以上示例数据中，若站2人一排，剩1人；若站3人一排，剩2人；若站5人一排，剩1人。那么总人数可能是11，也可能是41等。可以证明11是最小可能的人数。那么只需再找来5个人即可让同学们组成4×4的方阵。
