Description

这个东西…..你们懂的. 但是有一天,当佳姐兴致勃勃地和天游讲起<<非诚勿扰>>节目里的帅哥时,天游却送了佳姐两个字: “呵呵”。天游觉得这个节目里相亲的手段相当之业余. 于是向佳姐展示了ACM版的完美男女速配方案:

假设如今有N位男士和N位女士想要找到他们的合适伴侣。那么首先每位女士按照其对每位男士作为交往对象的偏爱程度给每位男士排名次，而每位男士也将对这N位女士如此做。这些名次将不准出现并列。然后每位男士将向心仪对象展示自己，经过残酷的竞争后各自找到合适的伴侣。

最开始，每位男士都没有被任何一位女士拒绝。求婚环节或许会经过很多轮，那么对于每一轮：

(1)    每位男士从还没有拒绝过自己的女士当中选出自己认为最理想的人，并向她表白。

 

(2)    每位女士在所有这一轮中从向她表白的男士当中选择自己认为最理想的一个，不答应也不拒绝。而把其余向她表白的男士拒绝掉。

 

这个简单的游戏经过若干轮后，幸运的事情将会发生！ 此时所有的女士恰好有一个追求者，而所有的男士也都找到一个心仪的对象。

 

天游解释了这个游戏的神奇之处：没有任意的两个配对，例如男士A和女士a，男士B和女士b，使得在A心目中b比a更理想，在b心目中A比B更理想（这不明摆着私奔么 - -）。因此天游说，这样的配对将非常之完美。（他知道，这样的完美结局是一定会出现的）

 

现在，假设你已经知道了这群男女间互相欣赏的高低程度序列，你能求出最终的结局吗？

Input

对于每组测试数据：

首先包括一个正整数N (1<=N<=1000)

              以下N*2行,每行有N个数字，第i+1行 (1<=i<=N ) 为编号为i的男士对女士们的      排序（第一个为最喜欢的）；第N+j+1行（1<=j<=N）则表示编号为j的女士对男             士们的排序，优先级同上。

N等于0的时候数据结束，0不做处理。

Output

对于每组测试数据，输出N行数字，每行包含一个整数，第i行的数字表示i号男子最终邂逅的佳人编号。

Sample Input

3<br/>1 2 3<br/>2 3 1<br/>2 1 3<br/>3 2 1<br/>2 3 1<br/>2 3 1<br/>0<br/>

Sample Output

3<br/>2<br/>1<br/>
