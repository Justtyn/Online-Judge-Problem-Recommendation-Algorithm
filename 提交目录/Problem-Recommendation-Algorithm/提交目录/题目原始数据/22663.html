Description

热身赛过程中因为俱乐部的注册比较麻烦，正好服务器又抽了一会，导致了大家的种种不愉快，Aem在这说声，抱歉。

        

其实比赛之后，Aem也有反省，如果我们主办方事先注册好一堆账号，情况就不同了。当然，注册一堆账号是麻烦的，所以有了批量注册的需求。

随后，Aem找俱乐部的管理员谈了谈，提了些建议。比如开发比赛专用临时账号的批量注册功能。（临时账号指不需要邮箱验证只能在比赛中使用的账号）

        

仔细想想，作为软件学院的学生，这么简单的东西，其实我们是这么该怎么做的。要开发一个批量注册账号功能，需要生成用户名列表模块，随机生成密码模块，以及控制用户生命周期模块（比赛结束后，用户失效）。

 

现在Aem委托你写生成用户名列表模块。

假定批量用户名为一个字符串和一个数组成。

给定一个字符串s，两个数a和b。要你输出s和k拼接而成的用户名。（a<=k<=b）


Input

第一行，一个字符串s。（长度不超过10，仅含有英文字母,区分大小写）

第二行，两个数a和b。（1<=a<=b<=100）


Output

若干行，每行一个用户名。（按字典序递增）


Sample Input

acmer
1 4

Sample Output

acmer1
acmer2
acmer3
acmer4
